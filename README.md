## 게임 기획 기본 템플릿

이 문서는 게임 기획 의사결정의 기준 문서입니다.
기획 변경이 생기면 아래 항목부터 업데이트합니다.

## 현재까지 기획 합의 요약
- 게임 방향: NPC 상호작용 중심의 스토리형 RPG
- 핵심 타겟: 20~30대 스토리 RPG 선호 캐주얼 유저
- 플레이 세션: 저녁 30분~1시간, 모바일 우선
- 핵심 감정: 성취감, 힐링, 설렘 등 인간적인 감정 경험
- 코어 플레이 톤: 언제든 멈추고 다시 이어도 부담 없는 진행

## 에이전트 작업 지침(현재 합의 반영)
- 항상 한국어로 답변한다.
- 수정 대상은 `main` 브랜치의 `README.md`를 우선한다.
- 항목별로 합의한 내용만 반영하고, 임의 확장은 하지 않는다.
- 변경 후에는 커밋 및 푸시 상태를 확인해 공유한다.

### 1) 게임 한 줄 정의
- 형식: "[누가] [언제] [무엇을] 하며 [어떤 재미]를 느끼는 게임"
- 예시: "바쁜 직장인이 10분 동안 리듬 전투로 성취감을 얻는 액션 로그라이크"
- 현재 정의: "캐주얼 유저가 저녁 30분~1시간 동안 NPC와 상호작용하며 퀘스트를 진행하고, 성취감·힐링·설렘 같은 인간적인 감정을 느끼는 스토리 중심 RPG"

### 2) 타겟 유저
- 핵심 타겟: 20~30대 스토리 RPG를 선호하는 캐주얼 유저
- 보조 타겟: 힐링/수집형 게임 유저, 감정선 중심 스토리를 선호하는 유저
- 플레이 성향(라이트/코어, 경쟁/협동, 스토리/시스템): 라이트 중심, 싱글 플레이, 스토리/시스템 균형
- 1회 평균 플레이 시간: 저녁 30분~1시간
- 주 플레이 플랫폼: 모바일(우선)

### 3) 핵심 재미 3기둥
- 기둥 1: 감정 몰입형 NPC 상호작용(대화/관계 변화가 스토리와 퀘스트 결과에 영향을 준다)
- 기둥 2: 편안하지만 분명한 퀘스트 성취감(수집/전달/만남 퀘스트를 부담 없이 완료하며 성취를 느낀다)
- 기둥 3: 다음 이야기가 궁금해지는 설렘(짧은 플레이에서도 다음 NPC 이벤트를 기대하게 만든다)
- 이 3가지가 약해지는 기능은 우선순위를 낮춘다.

### 4) 코어 루프
- 30초 루프(즉시 행동): 맵 이동 -> NPC와 짧은 대화/단서 확인 -> 다음 행동(이동/수집/전달) 결정
- 3분 루프(짧은 목표 달성): 소형 퀘스트 1개 진행(물건 찾기/전달/특정 NPC 만나기) -> 즉시 보상(대사 변화, 소량 재화, 관계도 소폭 상승)
- 30분 루프(세션 완결/성장 체감): 에피소드 단위 퀘스트 완료 -> 주요 감정 이벤트 발생 -> 관계도/해금/다음 화 예고 확인
- 루프 검증 질문: "지금 멈춰도 아쉽지 않고, 나중에 돌아와도 자연스럽게 이어서 플레이할 수 있는가?"

### 5) 성장/보상 구조
- 단기 보상(즉시 피드백): 퀘스트 단계 완료 시 감정 대사/표정 변화, 소량 재화, 관계도 소폭 상승 제공
- 중기 보상(세션 내 성장): 30~60분 플레이 안에서 에피소드 진행도 상승, 이동 지역/서브 퀘스트 해금, NPC 신뢰 단계 상승
- 장기 보상(계정/메타 성장): 누적 플레이로 스토리 아카이브(회상/도감) 확장, 주요 NPC 개인 스토리 라인 해금, 장기 업적 축적
- 희소 보상(특별 해금/수집): 특정 조건 달성 시 한정 이벤트 장면, 희귀 기념 아이템, 특별 코스튬/프로필 요소 해금

### 6) 의사결정 원칙
- 원칙 1: 감정의 진정성을 해치지 않는다. NPC 대사/연출/퀘스트 결과는 플레이어가 공감할 수 있는 감정선 안에서 설계한다.
- 원칙 2: 언제든 멈추고 다시 시작해도 불편하지 않아야 한다. 복귀 시 목표와 맥락을 짧고 명확하게 다시 보여준다.
- 원칙 3: 편안하지만 지루하지 않은 진행감을 유지한다. 조작/동선/정보 구조는 단순하게, 보상과 사건은 꾸준하게 제공한다.
- 충돌 시 우선순위 예시: 재미 > 가독성 > 리얼리티 > 볼륨
